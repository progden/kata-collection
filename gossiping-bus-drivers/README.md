# Gossiping Bus Drivers Kata

來源：[Kata-Log](https://kata-log.rocks/gossiping-bus-drivers-kata)

## 問題描述

公車司機們喜歡交換八卦，因此每當他們在公車站相遇時，會互相分享所知道的所有八卦。

每位司機在一天開始時都有一則八卦要分享。你的目標是找出**需要多少站（分鐘）才能讓所有司機知道所有八卦**。

## 規則

1. **路線循環**：每位司機有固定路線，會循環行駛
2. **初始八卦**：每位司機一開始擁有一則獨特的八卦
3. **時間計算**：每站之間行駛需要 1 分鐘
4. **八卦交換**：當兩位或多位司機在同一站相遇時，會立即分享所有已知的八卦
5. **工作日上限**：最多 480 分鐘（8 小時工作日）

## 輸入格式

路線以數字序列表示，例如：
```
3 1 2 3
3 2 3 1
4 2 3 4 5
```

每行代表一位司機的路線：
- 第一個數字是路線的站數
- 後續數字是依序經過的站牌編號

## 輸出格式

- 若所有司機能在 480 分鐘內知道所有八卦，回傳所需的分鐘數
- 若無法達成，回傳 `"never"`

## 範例

### 範例 1
```
輸入：
3 1 2 3
3 2 3 1
4 2 3 4 5

輸出：5
```

### 範例 2
```
輸入：
2 1 2
5 2 8

輸出：never
```

## TDD 測試案例建議 (ZOMBIES 順序)

1. **Zero**: 只有一位司機 → 回傳 1
2. **One**: 兩位司機第一站就相遇 → 回傳 1
3. **Many**: 多位司機需要多次相遇
4. **Boundary**: 剛好第 480 分鐘完成 / 超過 480 分鐘
5. **Exception**: 永遠無法相遇 → "never"
